# AAI to XYZ Governance Mapping

## Purpose

This document maps the Aging Aircraft Index (AAI) agentic system to the XYZ Decision Space Model.

The goal is to show how AAI data elements should be treated by different agents before reasoning begins.

---

## Governance Principle

AAI depends on a clear distinction between:

1. **source evidence** — raw records that must be preserved;
2. **anchored context** — authoritative or semi-authoritative information that may be interpreted but not silently rewritten;
3. **derived structure** — normalized records, classifications, and clusters generated by agents;
4. **analytical conclusions** — health scores, trends, recommendations, and report summaries;
5. **temporary working state** — scratchpad or intermediate reasoning that should not be passed downstream unless promoted.

XYZ governance assigns each of these elements a posture and treatment rule.

---

## Key AAI Elements

| Element | Description | Typical Posture |
|---|---|---|
| Raw TOD machine log | Machine-generated aircraft report | IMMUTABLE |
| Pilot EOD report | Pilot-submitted narrative observation | ANCHORED |
| Maintenance EOD report | Maintenance-submitted action/status note | ANCHORED |
| Source timestamp | Date/time or reporting period of source record | IMMUTABLE |
| Aircraft identifier | Aircraft registration or fleet identifier | IMMUTABLE |
| Message ID | Machine-generated system message identifier | IMMUTABLE |
| ATA classification | System classification, sometimes inferred | NEGOTIABLE |
| Normalized event record | Structured event created by agent | AGENT_OWNED |
| Deduplication link | Agent-created link between related records | AGENT_OWNED |
| Cluster ID | Agent-created reliability cluster identifier | AGENT_OWNED |
| Cluster summary | Agent-generated summary of related events | NEGOTIABLE |
| Discrepancy status | Active, resolved, monitoring, or unknown | ANCHORED or NEGOTIABLE |
| Health score | Agent-generated aircraft health score | AGENT_OWNED |
| Final report | Published AAI output | ANCHORED |

## Enforcement Rules

### Integrity Checks

Required for:

- raw TOD records;
- source timestamps;
- aircraft identifiers;
- message IDs;
- source record IDs;
- evidence links.

### Semantic Compliance Checks

Required for:

- no source rewriting;
- no source omission;
- no unsupported resolution claim;
- no unsupported cluster promotion;
- no health score without evidence;
- no treating inferred status as maintenance-confirmed status.

---

## Example Compliance Rules

| Rule | Description |
|---|---|
| `no_source_rewrite` | Raw source text must not be altered |
| `preserve_source_link` | Derived records must retain source references |
| `no_unsupported_resolution` | Discrepancy cannot be marked resolved without maintenance evidence |
| `flag_inferred_ata` | Inferred ATA classification must be labeled as inferred |
| `evidence_required_for_score` | Health score must include supporting clusters and source references |
| `no_silent_cluster_merge` | Cluster merges must preserve prior cluster IDs and rationale |

---

## Governance Pressure

The highest governance pressure in AAI is expected around:

- health scoring;
- discrepancy status interpretation;
- maintenance resolution assessment;
- report publication;
- cluster promotion from weak signals to active monitoring items.

These are the areas where the system should use verification gates, audit logging, and human review.

---

## Summary

AAI is a strong application for the XYZ Decision Space Model because it contains multiple data types with different authority relationships:

- raw evidence;
- authoritative maintenance context;
- pilot observations;
- agent-generated structure;
- inferred clusters;
- analytical scores;
- final reports.

XYZ makes those distinctions explicit and enforceable before agent reasoning begins.
